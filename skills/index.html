My Skills

  • Modeling and Simulation
    As an engineer and researcher, I develop models to represent social and industrial systems. When simulated, these models give insight by predicting future states or explaining how would system behave under certain conditions. More importantly, these benefits are provided in a safe-to-test and cost effective manner. In other words, one can test extreme conditions in a model without damaging the actual system. Among many modeling paradigms, I am particularly interested in Agent-Based Modeling (ABM). The reason is that, ABM allows to represent a system based on its constituting units - agents. With agents, it is possible to model a society, for instance, according its real heterogeneity whereas modeling paradigms such as system dynamics assume that all individuals as same. Within ABM, I majorly focused on combining big data and machine learning in model-building process to increase empirical grounding in the model. I believe such data-driven approaches will dominate the field in the future.

  • Data Mining and Machine Learning
    In the era of Big Data, data mining and machine learning became defacto tools to gain insight from large datasets. It has been nine years since I developed my first machine learning model (OCR application using neural networks). Since then, I mostly worked on data gathered from the real-world such as blogs and social media. The goal is often finding patterns in these data. Often times, I develop supervised machine learning models by creating a training set (ground truth) and testing the rest of the data.

  • Big Data Proccessing
    I have made significant effors in the last few years to learn about Big Data processing environments. Particularly, I gained experience on working Hadoop environment. I use Hadoop HDFS to store my Twitter data. I use Hive to process this stored data which contains more than half a billion records. When the built-in functionalities of Hive fall short, I develop custom UDFs to extend the capabilities of Hive. I am planning to start using Spark to develop and run my data mining and machine learning models on a distibuted environment.

While not my prominent strength, I am eager to extend my knowledge in statistical analysis and modeling if my problems require.

  • Databases
    I interact with database systems on a daily basis and work with traditional SQL-oriented databases for the last ten years. I previously worked on SQL Server and Oracle database systems. I currently work on PostgreSQL with PostGIS extension. Specifically, I use PostgreSQL+PostGIS to store my location based tweet and venue data. In the last five years, I have experimented with No-SQL databases. I used Redis as caching database for M&S Cube app. I also used MongoDB to store tweets in their native JSON format. My preference is to use a database system that fits my needs.
